MinIO 停止更新了，虽然目前还能用，但也提前了解下替代品。我主要用来做开发测试，所以要求有 Docker 镜像，启动快，依赖简单，可以单节点部署。

| 名称                  | S3 兼容性              | 单节点部署难易度           | 维护活跃度                        | 作者核心评价与优缺点                                                                 | 备注                              |
|---------------------------|------------------------|----------------------------|-----------------------------------|--------------------------------------------------------------------------------------|-----------------------------------|
| **S3Proxy**               | 是（部分限制见文档）   | 非常简单（Docker 直接替换） | 长期维护，单维护者，部分依赖已归档 | 极轻量、配置简单、镜像小、长期存在；最轻量级选项之一                                 | 追求极简首选                      |
| **RustFS**                | 是（文档声称完整）     | 简单（Docker 切换方便）     | 新项目（2024 年启动），活跃但 alpha 阶段 | 新项目、界面漂亮、有 GUI，但 alpha 阶段且近期有安全漏洞                             | 未来潜力大，现阶段需谨慎          |
| **SeaweedFS**             | 是                     | 简单～中等（需少量 auth 配置，但即将简化） | 长期活跃（2012 年起），单主要维护者，高 Docker pulls | 成熟、带基础 UI、快速上手、即将移除复杂认证；目前最接近 MinIO 体验的替代             | 作者目前主力考虑方向              |
| **CloudServer** (Zenko / Scality) | 是（完整）             | 中等（Docker 可替换，但文档/镜像略旧） | 活跃，多贡献者，高 pulls（部分 tag 已过时） | 功能完整、有商业背景，但属于 Zenko 套件、命名稍混乱、部署略有额外复杂度             | 可接受，但有套件 overhead         |
| **Garage**                | 是                     | 复杂（需 TOML 配置 + init 容器） | 活跃，多贡献者，有资助            | 设计偏分布式、单节点配置繁琐、key 格式严格；对快速 demo 来说过于复杂                | 不适合单节点快速测试              |
| **Apache Ozone**          | 是                     | 极复杂（至少 4 节点，Hadoop 生态） | Apache 基金会治理，多贡献者       | 企业级重量级方案、非轻量容器设计；完全不适合本地单节点场景                          | 完全不推荐用于本地开发            |

### 推荐
- SeaweedFS → 成熟度高、体验最接近 MinIO、即将进一步简化配置。
- S3Proxy → 极致轻量、几乎零配置负担。

### 不推荐
- RustFS → 还未发布正式版。
- CloudServer → 功能复杂。
- Garage → 配置复杂。
- Apache Ozone → 太笨重。

### 不满足要求方案
- JuiceF → 本质是 POSIX 分布式文件系统，其 S3 Gateway 基于 MinIO 实现。
- Rook → Kubernetes 的 Ceph 等存储编排器（Operator），本身不直接提供 S3 服务，而是管理集群级存储。
- Ceph RGW → 企业级分布式存储平台，设计目标是多节点高可用/自愈。
- Versitygw → 高性能 S3 网关（gateway），将 S3 API 映射到 POSIX 文件系统或其他后端。
